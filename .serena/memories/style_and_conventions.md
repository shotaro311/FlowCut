# コードスタイル/規約（現状把握）

## 共通
- 変更は小さく分けて行い、不要なリネーム/大規模整形は避ける。
- コメントは最小限（コードから読める説明は書かない）。

## Python
- 型ヒントを優先（`typing` / `| None` など）。
- 例外メッセージは日本語で簡潔に。
- ログは `logger` を使用し、ユーザー向けUIとは分離。

## TypeScript/JavaScript（もし触る場合）
- 2スペース、シングルクオート、セミコロン必須（リポジトリガイドライン）。

## ドキュメント運用
- PLAN: `docs/plan/` の最新ファイルを更新（新規 PLAN を増やさない）。
- 要件: `docs/requirement.md` と実装のズレが出たら同期。
